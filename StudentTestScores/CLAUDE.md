## Project — StudentTestScores (Kaggle PS S6E1)
[`playground-series-s6e1`](https://www.kaggle.com/competitions/playground-series-s6e1):
**regression** predicting a student's continuous `exam_score` (range ~19.6–100, mean 62.5, std 18.9)
from study/lifestyle features. Train 630,000 rows · test 270,000 rows.

**Schema (confirmed from raw CSVs):** 11 features, **no missing values**, no test-only categorical
levels. Numeric (4): `age` (int 17–24), `study_hours`, `class_attendance`, `sleep_hours` (float).
Categorical (7, all low-cardinality): `gender` (3), `course` (7), `internet_access` (2),
`sleep_quality` (3), `study_method` (5), `facility_rating` (3), `exam_difficulty` (3). Native GBDT
categorical handling is fine; target encoding is unlikely to help. Submission columns `id,exam_score`.

**Metric: RMSE** (lower is better). `src/metric.py` is the single source of truth. OOF and blend
comparisons both go through `rmse()` so we optimise exactly what Kaggle grades. Naive floor (predict the
train mean) ≈ target std ≈ 18.9; public LB tops out ~8.57.

**Character:** clean synthetic dataset. Gate every FE/tuning change on OOF RMSE and keep only
above-noise gains; do not chase noise (the sibling `OptimalFertilizer`/`StudentHealthRisk` lesson: FE
tends to be neutral/negative on these synthetic datasets).

## Environment & commands
Uses **uv** (not pip/`requirements.txt`). Deps pinned in `pyproject.toml` / `uv.lock`. **Run everything
from this project dir (`StudentTestScores/`)** and prefix Python with `uv run`:

```bash
uv sync                                          # create/refresh the venv
uv run pytest                                    # RMSE + schema/feature tests
uv run python -m src.train --model lgbm          # 5-fold CV -> OOF + bagged test arrays
uv run python -m src.blend                       # equal-weight average of base learners
uv run python -m src.submit --model blend        # build + validate submission
```

**Kaggle auth uses a `KGAT_` token at `~/.kaggle/access_token` (no `kaggle.json`).** The CLI reads it
from an env var:

```bash
export KAGGLE_API_TOKEN="$(cat ~/.kaggle/access_token)"
uv run kaggle competitions download -c playground-series-s6e1 -p data/raw
unzip -o data/raw/playground-series-s6e1.zip -d data/raw
```

## Architecture (prediction-file pipeline — mirrors sibling `OptimalFertilizer/`)
`src/` communicates through **saved prediction arrays** so training and submission are decoupled and any
subset of models can be blended after the fact. Each base learner writes a 1-D `{model}_oof.npy`
(shape `(n_train,)`) + fold-bagged `{model}_test.npy` (shape `(n_test,)`) + `{model}_metrics.json` to
`experiments/artifacts/`, alongside shared `y.npy` (float target) and `test_id.npy`.
- `src/data.py` — schema constants (`TARGET`, `CATEGORICAL`, `NUMERIC`, `ID`, `TARGET_MIN/MAX`) +
  loaders (raise on schema mismatch); cache to `data/processed/`; `load_sample_submission()`.
- `src/cv.py` — `KFold(n_splits=5, shuffle=True, random_state=42)` (plain; rows are i.i.d. synthetic).
- `src/metric.py` — `rmse(y_true, y_pred)`.
- `src/features.py` — leakage-safe FE (raw passthrough `base` baseline; categoricals as `category`
  dtype with train/test level alignment; optional `ratios` group, off until it beats OOF).
- `src/train.py` — regression CV loop scoring OOF RMSE; saves OOF + bagged test vectors. Models:
  LightGBM / XGBoost / CatBoost (`--model lgbm|xgb|cat`).
- `src/blend.py` — equal-weight average of base learners' prediction vectors, scored on OOF RMSE.
- `src/submit.py` — clip to `[TARGET_MIN, TARGET_MAX]`, build + validate submission (row count, ids,
  finite/in-range values, columns `id,exam_score`) vs `sample_submission.csv`; optional upload `--submit`.

Git-ignored: `data/`, `models/`, `outputs/`, `experiments/artifacts/`, `reports/figures/`, `.venv/`.

## EDA
`notebooks/01_eda.ipynb` is **generated by a build script** (`notebooks/eda_build.py` using `nbformat`),
then executed with `uv run jupyter nbconvert --to notebook --execute --inplace`. Revise the generator and
re-run rather than hand-editing notebook JSON. Figures → `reports/figures/`; conclusions →
`reports/EDA_FINDINGS.md`.

## Conventions & hard-won lessons
- **Tune/compare on OOF RMSE**, the real metric. Treat sub-noise OOF differences as ties.
- Expect a CV↔LB gap; an **equal-weight diverse blend** tends to generalize better than OOF-optimized
  weighting.
- **Show CV/OOF metrics and get explicit approval before any Kaggle submission.** Log every run in
  `experiments/README.md`. Commit incrementally after each meaningful step; stage only this project's
  files (the monorepo contains unrelated sibling projects).
- The competition deadline (2026-01-31) has passed, but Kaggle Playground still scores late submissions.
