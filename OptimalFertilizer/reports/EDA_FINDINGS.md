# EDA Findings — Predicting Optimal Fertilizers (Kaggle PS S5E6)

Source notebook: [`notebooks/01_eda.ipynb`](../notebooks/01_eda.ipynb) (generated by
[`notebooks/eda_build.py`](../notebooks/eda_build.py), executed; figures in `reports/figures/`).
Competition: [`playground-series-s5e6`](https://www.kaggle.com/competitions/playground-series-s5e6).
Metric: **MAP@3** (submit up to 3 class names per row, best-first; per-row score `1/rank` if the truth
is in the top-3, else 0).

## Headline

This is a **near-noise synthetic dataset**. The features barely predict the fertilizer: mutual
information is ~0 for every column, all pairwise numeric correlations are ~0, and even conditioning on
the full `Soil × Crop` grid lifts in-sample MAP@3 only **+0.011 over the naive prior**. The floor to
beat (predict the 3 globally most-frequent classes for everyone) is **MAP@3 = 0.2785**; a good GBDT
that exploits joint interactions across all 8 features should land only modestly above it (~0.33–0.34
OOF expected). **Ranking quality and top-3 tie-handling dominate; treat any sub-0.001 OOF change as
noise.** Good news: train and test are the **same distribution** (no drift), so CV should track the LB.

## Dataset overview

- **Train:** 750,000 rows (`id` 0–749,999). **Test:** 250,000 rows (`id` 750,000–999,999).
  `sample_submission` = 250,000 rows, columns `id, Fertilizer Name`.
- **8 features**, all clean integers, no missing values in either split:
  - env numeric: `Temparature` *(sic — misspelled in the raw data)*, `Humidity`, `Moisture`
  - categorical: `Soil Type` (5 levels), `Crop Type` (11 levels)
  - NPK numeric: `Nitrogen`, `Potassium`, `Phosphorous`
- Numeric ranges are tight and bounded: Temp 25–38, Humidity 50–72, Moisture 25–65, N 4–42, K 0–19,
  P 0–42 (see `figures/numeric_distributions.png`).

## Class balance

- 7 classes, **mildly imbalanced** (max/min ratio **1.24**): `14-35-14` 15.3%, `10-26-26` 15.2%,
  `17-17-17` 15.0%, `28-28` 14.8%, `20-20` 14.8%, `DAP` 12.7%, `Urea` 12.3%.
  (`figures/class_balance.png`)
- **Naive-prior MAP@3 = 0.2785** — predicting the 3 most-frequent classes (`14-35-14 10-26-26 17-17-17`)
  for every row. This is the baseline floor and the number the naive submission should score on the LB.

## Feature signal (the crux)

Three views, all pointing the same way — **very weak marginal signal**:

1. **Mutual information** `I(feature; target)` is tiny for all features (nats):
   Moisture 0.0062 (highest), Nitrogen 0.0027, Temparature 0.0027, Crop Type 0.0021, Humidity 0.0020,
   Phosphorous 0.0011, Soil Type 0.0007, **Potassium ≈ 0.0000**. (`figures/mutual_information.png`)
2. **Per-class z-scored numeric means** do differ across classes (spread ~2.4–3.1 σ per feature;
   `figures/per_class_means.png`), but within-class variance dwarfs the between-class mean gaps — hence
   the near-zero MI. The class differences are real but only exploitable *jointly*, not marginally.
3. **Single-feature MAP@3 baselines** (rank classes by `P(class | feature)`, in-sample):

   | signal | train MAP@3 | lift vs prior |
   |--------|------------:|--------------:|
   | Soil × Crop (joint, 55 combos) | 0.2897 | **+0.0112** |
   | Moisture (10 bins) | 0.2851 | +0.0066 |
   | Crop Type | 0.2840 | +0.0055 |
   | Potassium / Phosphorous / Humidity / Temp / Nitrogen (10 bins) | ~0.282–0.283 | +0.004–0.005 |
   | Soil Type | 0.2814 | +0.0029 |
   | *(naive prior)* | 0.2785 | 0 |

   These are in-sample upper bounds. **`Soil × Crop` is the strongest single signal** → the best FE
   candidate. A GBDT combining all 8 features with fine-grained interactions will exceed any of these,
   which is exactly why trees (not a linear model) are required.

## Data quality

- **No missing values**; **no exact-duplicate feature rows**.
- **Every one of the 750,000 feature vectors is unique, and none maps to conflicting labels** — so
  there is no exact-duplicate label-conflict ceiling. But because the marginal/joint signal is so weak,
  memorizing full feature vectors does not generalize; the generalizable signal lives in coarse joint
  structure, not in exact matches.

## Train vs test drift

- **None.** Numeric KS train-vs-test p-values all ≫ 0.01 (Temp 0.43, Humidity 0.78, Moisture 1.00,
  Nitrogen 0.13, Potassium 0.91, Phosphorous 0.05 — 0/6 significant at p<0.01). Categorical share
  deltas < 0.15 pp for both `Soil Type` and `Crop Type`. (`figures/train_test_drift.png`)
- **Implication:** i.i.d. synthetic rows, identical train/test distribution → **`StratifiedKFold` on
  the target is appropriate and CV should track the leaderboard well** (unlike sibling EnergyAnomaly,
  which needed grouped CV).

## Correlation / NPK structure

- Max |off-diagonal| numeric correlation = **0.01** — features are mutually independent; **no
  redundancy and no obvious NPK relationship** to exploit linearly (`figures/numeric_correlation.png`).
  Engineered NPK ratios/sums are worth a try but expectations are low.

## Augmentation verdict

The competition data was synthesized from the **UCI "Fertilizer Prediction"** dataset
(`gdabhishek/fertilizer-prediction`, 99 rows), confirmed by:

- **Identical schema** (same misspelled `Temparature`; `Humidity ` has a trailing space in the
  original), **identical 7 classes, 5 soil types, 11 crop types** (zero set difference), and
  **identical numeric min/max ranges** — the generator sampled within the original's observed support.
- Original class balance differs (Urea most common there, least common in the competition).

**Verdict: worth *testing* in Step 4, but expect it to be neutral.** 99 rows is 0.013% of the training
set; the only reason to try is that it is the "ground-truth" seed. Low priority; adopt only if OOF
MAP@3 improves above noise. Stored (git-ignored) at `data/original/Fertilizer Prediction.csv`.

## Modeling implications (feeds the plan)

1. **CV = `StratifiedKFold(5, shuffle=True, random_state=42)`** on the target — no drift, i.i.d. rows,
   CV trustworthy. (No grouping needed.)
2. **GBDT ensemble over a linear model** — signal is weak, nonlinear, and only joint; trees capturing
   interactions (esp. `Soil×Crop` and numeric×categorical) are the right tool.
3. **MAP@3 is a ranking metric** — the job is well-*ranked* per-class probabilities, then top-3 argsort.
   Because even the best conditional barely beats the prior, the top-3 will usually include the
   globally-frequent classes; the model mostly re-orders them per row.
4. **FE priorities:** `Soil×Crop` combined categorical first (highest measured lift); then NPK
   ratios/sums/differences and env bins — each gated on OOF MAP@3, keep only above-noise.
5. **Target encoding likely won't help** — categoricals are low-cardinality and signal is weak, so TE
   mostly adds fold variance (the EnergyAnomaly lesson). Keep it behind a flag, off by default.
6. **Do not chase noise.** Naive floor 0.2785; expect OOF ~0.33–0.34. Sub-0.001 differences are ties;
   an equal-weight diverse blend is the safe bet, and every change must clear OOF MAP@3.
