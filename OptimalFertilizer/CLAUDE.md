## Project — OptimalFertilizer (Kaggle PS S5E6)
[`playground-series-s5e6`](https://www.kaggle.com/competitions/playground-series-s5e6):
**multi-class classification** recommending the best `Fertilizer Name` (7 classes:
`Urea, DAP, 14-35-14, 28-28, 17-17-17, 20-20, 10-26-26`) from soil/crop/weather conditions.
Train 750,000 rows · test 250,000 rows. 8 features: `Temparature, Humidity, Moisture` (env numeric),
`Soil Type`, `Crop Type` (categorical), `Nitrogen, Potassium, Phosphorous` (NPK numeric).

**Schema gotchas (confirmed from raw CSVs):** the temperature column is misspelled **`Temparature`**
(no "e" — use the literal column name in code). All six numeric features are **`int64`**. Categoricals
are **low-cardinality**: `Soil Type` (5 levels: Black/Clayey/Loamy/Red/Sandy), `Crop Type` (11 levels)
→ `Soil×Crop` = 55 combos; native GBDT categorical handling is fine, target encoding is unlikely to
help much. No missing values in either split. Target is mildly imbalanced (Urea/DAP ~92–95k vs others
~111–114k).

**Metric: MAP@3.** Submission is `id,Fertilizer Name` with **up to 3 space-separated class names per
row, best-first**. With one true label per row, per-row score is `1/rank` if the truth is in the top 3
(1, 1/2, or 1/3), else 0. → the model must output well-*ranked* class probabilities; we take the top-3
argsort. `src/metric.py` is the single source of truth for both scoring and building the 3-name string.

**Character:** notoriously **low-signal** synthetic dataset — features weakly predict the target and
public LB MAP@3 clusters tightly (~0.36–0.38). Ranking quality and top-3 tie-handling dominate. Gate
every FE/tuning change on OOF MAP@3 and keep only above-noise gains; do not chase noise.

## Environment & commands
Uses **uv** (not pip/`requirements.txt`). Deps pinned in `pyproject.toml` / `uv.lock`. **Run everything
from this project dir (`OptimalFertilizer/`)** and prefix Python with `uv run`:

```bash
uv sync                                          # create/refresh the venv
uv run pytest                                    # MAP@3 + submission-format tests
uv run python -m src.train --model lgbm          # 5-fold CV -> OOF + bagged test arrays
uv run python -m src.blend                       # rank-average blend of base learners
uv run python -m src.submit --model blend        # build + validate submission
```

**Kaggle auth uses a `KGAT_` token at `~/.kaggle/access_token` (no `kaggle.json`).** The CLI reads it
from an env var:

```bash
export KAGGLE_API_TOKEN="$(cat ~/.kaggle/access_token)"
uv run kaggle competitions download -c playground-series-s5e6 -p data/raw
unzip -o data/raw/playground-series-s5e6.zip -d data/raw
```

## Architecture (prediction-file pipeline — mirrors sibling `EnergyAnomaly/`)
`src/` communicates through **saved prediction arrays** so training and submission are decoupled and any
subset of models can be blended after the fact. Each base learner writes `{model}_oof.npy` (OOF
probability matrix, shape `(n_train, 7)`) + fold-bagged `{model}_test.npy` (shape `(n_test, 7)`) +
`{model}_metrics.json` to `experiments/artifacts/`, alongside shared `y.npy`, `classes.npy`,
`test_id.npy`.
- `src/data.py` — schema constants (`TARGET`, `CATEGORICAL`, `NUMERIC`, `ID`) + loaders (raise on schema
  mismatch); cache to `data/processed/`; `load_sample_submission()`.
- `src/cv.py` — `StratifiedKFold(n_splits=5, shuffle=True, random_state=42)` on the target.
- `src/metric.py` — `mapk3(y_true, proba, classes)` and the top-3 submission-string builder.
- `src/features.py` — leakage-safe FE (raw passthrough baseline first; categoricals as `category` dtype
  with train/test level alignment; later NPK ratios/sums, env bins, Soil×Crop combo).
- `src/target_encoding.py` — in-fold (leak-safe) target/count encoding, off by default until it beats OOF.
- `src/train.py` — multiclass CV loop scoring OOF MAP@3; saves OOF + bagged test arrays. Models:
  LightGBM / XGBoost / CatBoost (`--model lgbm|xgb|cat`).
- `src/blend.py` — equal-weight **rank-average** of base learners (prefer over OOF-optimized stacking).
- `src/submit.py` — build + validate submission (row count, ids, exactly 3 valid distinct class names,
  columns `id,Fertilizer Name`) vs `sample_submission.csv`; optional Kaggle upload with `--submit`.

Git-ignored: `data/`, `models/`, `outputs/`, `experiments/artifacts/`, `reports/figures/`, `.venv/`.

## Conventions & hard-won lessons
- **Tune/compare on the real metric.** MAP@3 is rank-based; log-loss improvements don't always transfer.
  Always score candidates on OOF MAP@3, not just the training objective.
- Expect a CV↔LB gap; an **equal-weight diverse blend** tends to generalize better than OOF-optimized
  weighting (sibling-project lesson). Treat sub-noise OOF differences as ties.
- **Show CV/OOF metrics and get explicit approval before any Kaggle submission.** Log every run in
  `experiments/README.md`. Commit incrementally after each meaningful step; stage only this project's
  files (the monorepo contains unrelated sibling projects).
