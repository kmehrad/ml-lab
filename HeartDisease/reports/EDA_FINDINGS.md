# EDA Findings — HeartDisease (PS S6E2)

See `notebooks/01_eda.ipynb` (generated by `notebooks/eda_build.py`) for the full run;
figures in `reports/figures/`.

## Data quality
- **630,000 train / 270,000 test rows.** Zero missing cells in either split. Zero
  duplicate feature rows in train or test.
- **No train/test drift:** numeric feature means match to ≤0.04 absolute; categorical
  level-share differences are all ≤0.31 percentage points. A plain `StratifiedKFold(5)`
  should track the leaderboard well.

## Target
- `Heart Disease`: `Absence` 347,546 (55.2%) / `Presence` 282,454 (44.8%) — **near-balanced**,
  no resampling needed.

`Thallium` is the strongest single feature (AUC 0.80 alone). `BP` and `FBS over 120` are
essentially inert (AUC ≈ 0.51, near coin-flip) — candidates to drop if they ever hurt a
model, though GBDTs handle irrelevant features natively so this is low priority.

## Implication for modeling
- Public leaderboard top ≈ 0.9553–0.9554 AUC; notebook titles suggest a plain GBDT k-fold
  baseline already lands near that ceiling — **less FE/tuning headroom than the harder
  sibling projects** (OptimalFertilizer, StudentHealthRisk). Expect diminishing returns
  from feature engineering; gate every change on OOF AUC and don't over-invest if GBDTs
  are already near the public ceiling.
- Candidate interaction terms for the `interactions` feature group (only keep if they beat
  OOF AUC): `Thallium × Chest pain type` (two strongest categoricals), `ST depression ×
  Slope of ST` (related ECG stress-test signals).
- Original 270-row UCI Statlog Heart data (`ritwikb3/heart-disease-statlog` on Kaggle) is
  schema-compatible and worth a quick augmentation test, though prior siblings found this
  neutral-to-negative when the synthetic data is already clean and large.
