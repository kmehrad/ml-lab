## Project — HeartDisease (Kaggle PS S6E2)
[`playground-series-s6e2`](https://www.kaggle.com/competitions/playground-series-s6e2):
**binary classification** predicting `Heart Disease` (`Presence`/`Absence`) from the classic
UCI Statlog Heart features. Train 630,000 rows · test 270,000 rows.

**Schema (confirmed from raw CSVs):** 13 features, **no missing values, no duplicate rows,
no train/test drift**. Numeric (5): `Age`, `BP`, `Cholesterol`, `Max HR` (int), `ST depression`
(float). Categorical/ordinal-int (8): `Sex` (2), `Chest pain type` (4), `FBS over 120` (2),
`EKG results` (3), `Exercise angina` (2), `Slope of ST` (3), `Number of vessels fluro` (4),
`Thallium` (3 levels, coded 3/6/7). Target encoded `Presence=1`; positive rate 44.8% (near-
balanced). Submission columns `id,Heart Disease` — **probability of `Presence`**, not the
string label (`sample_submission.csv` ships a numeric placeholder, and AUC needs continuous
scores).

**Metric: ROC AUC** (higher is better; confirmed via Kaggle API `evaluationMetric: "Roc Auc
Score"`, not visible on the JS-rendered web page). `src/metric.py` is the single source of
truth. Threshold-free ranking metric — unlike StudentHealthRisk's balanced accuracy, there is
no post-hoc decision-rule tuning step.

**Character:** high-signal dataset (unlike OptimalFertilizer) — several features correlate
0.4–0.6 with the target (`Thallium` 0.606, `Chest pain type` 0.461, `Exercise angina` 0.442,
`Number of vessels fluro` 0.439, `ST depression` 0.431, `Slope of ST` 0.415, `Max HR` −0.441,
`Sex` 0.342; `BP`/`FBS over 120` ≈ inert). Live LB top ≈ 0.9553–0.9554; public notebooks show
plain GBDT k-fold baselines already near that ceiling, so GBDTs are likely close to optimal
here (less FE/tuning headroom than the harder siblings). Original 270-row UCI Statlog Heart
data is available on Kaggle as `ritwikb3/heart-disease-statlog` for augmentation testing.

## Environment & commands
Uses **uv** (not pip/`requirements.txt`). Deps pinned in `pyproject.toml` / `uv.lock`. **Run
everything from this project dir (`HeartDisease/`)** and prefix Python with `uv run`:

```bash
uv sync                                          # create/refresh the venv
uv run pytest                                    # AUC + schema/submission-format tests
uv run python -m src.train --model lgbm          # 5-fold CV -> OOF + bagged test arrays
uv run python -m src.blend                       # equal-weight average of base learners
uv run python -m src.submit --model blend        # build + validate submission
```

```bash
kaggle competitions download -c playground-series-s6e2 -p data/raw
unzip -o data/raw/playground-series-s6e2.zip -d data/raw
```

## Architecture (prediction-file pipeline — mirrors sibling `StudentTestScores/`)
`src/` communicates through **saved prediction arrays** so training and submission are
decoupled and any subset of models can be blended after the fact. Each base learner writes a
1-D `{model}_oof.npy` (shape `(n_train,)`, predicted probability of `Presence`) + fold-bagged
`{model}_test.npy` (shape `(n_test,)`) + `{model}_metrics.json` to `experiments/artifacts/`,
alongside shared `y.npy` (0/1 target) and `test_id.npy`.
- `src/data.py` — schema constants (`TARGET`, `CATEGORICAL`, `NUMERIC`, `ID`) + loaders (raise
  on schema mismatch); cache to `data/processed/`; `load_sample_submission()`; target mapped
  to `y = (df[TARGET] == "Presence").astype(int)`.
- `src/cv.py` — `StratifiedKFold(n_splits=5, shuffle=True, random_state=42)` on `y`.
- `src/metric.py` — `roc_auc(y_true, y_prob)` wrapping `sklearn.metrics.roc_auc_score`.
- `src/features.py` — leakage-safe FE (raw passthrough `base` baseline; categoricals as
  `category` dtype; candidate interaction terms gated on OOF AUC).
- `src/train.py` — binary CV loop scoring OOF AUC; saves OOF + bagged test probability
  vectors. Models: LightGBM / XGBoost / CatBoost (`--model lgbm|xgb|cat`).
- `src/blend.py` — equal-weight average of base learners' predicted probabilities (AUC is
  rank-based, so plain averaging across similarly-calibrated GBDTs is standard).
- `src/submit.py` — build + validate submission (row count, ids, values in `[0,1]`, columns
  `id,Heart Disease`) vs `sample_submission.csv`; optional upload `--submit`.

Git-ignored: `data/`, `models/`, `outputs/`, `experiments/artifacts/`, `reports/figures/`,
`.venv/`.

## EDA
`notebooks/01_eda.ipynb` is **generated by a build script** (`notebooks/eda_build.py` using
`nbformat`), then executed with `uv run jupyter nbconvert --to notebook --execute --inplace`.
Revise the generator and re-run rather than hand-editing notebook JSON. Figures →
`reports/figures/`; conclusions → `reports/EDA_FINDINGS.md`.

## Conventions & hard-won lessons
- **Tune/compare on OOF AUC**, the real metric. Treat sub-noise OOF differences as ties.
- Expect a CV↔LB gap; an **equal-weight diverse blend** tends to generalize better than
  OOF-optimized weighting (sibling-project lesson).
- **Show CV/OOF metrics and get explicit approval before any Kaggle submission.** Log every
  run in `experiments/README.md`. Commit incrementally after each meaningful step; stage only
  this project's files (the monorepo contains unrelated sibling projects).
- Plain MLP tends to be too weak to help a GBDT blend (StudentHealthRisk lesson); if GBDTs
  plateau, cross-family NNs (TabM/FTT via pytabkit) are the tested lever for genuine
  decorrelation, not vanilla MLP.
- The competition deadline (2026-02-28) has passed, but Kaggle Playground still scores late
  submissions.
