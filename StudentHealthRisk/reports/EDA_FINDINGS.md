# EDA findings — StudentHealthRisk (PS S6E7)

Generated by `notebooks/eda_build.py` → `notebooks/01_eda.ipynb`; figures in `reports/figures/`.
Metric is **Balanced Accuracy Score** (mean per-class recall); the naive "always-majority" floor is
**0.333**.

## Headlines
- **Strong, learnable signal.** A single feature, `stress_level`, already reaches **0.754** train
  balanced accuracy; `sleep_duration` (binned) 0.665; `physical_activity_level` 0.550. The full model
  should sit far above the floor (a 40k-sample LGBM smoke test already hit **0.946** tuned OOF).
- **CV will track the LB.** No train/test drift: numeric KS p-values all > 0.2 (0/7 below 0.01), and
  categorical share deltas are tiny (gender 3.4pp is the largest, but gender has ~0 signal). Missing
  rates are identical across train/test. → **StratifiedKFold(5) on the target is the right scheme**;
  no GroupKFold needed.
- **Clean labels, high ceiling.** 0 exact-duplicate feature rows and 0 feature vectors mapping to >1
  class — no irreducible label ambiguity, so the signal ceiling is high.

## 1. Class balance
`at-risk` 85.87% · `unhealthy` 8.36% · `fit` 5.77% (max/min ratio 14.9). Heavy imbalance is the whole
game: under balanced accuracy every class counts equally, so minority recall (fit, unhealthy) dominates
the score. GBDTs use `class_weight="balanced"`; the decision is then re-tuned on OOF (`metric.tune_weights`).

## 2–3. Distributions
Numeric features are smooth, plausible physiological ranges (bmi 16–35, heart_rate 50–108, sleep 3–10h,
steps 1k–15k). All six categoricals are 3-level and near-uniform. Max numeric correlation is only 0.438
— little redundancy.

## 4. Missingness — partially informative
Missing values in **every** column; **49.3% of rows have ≥1 missing**. Highest rates: `stress_level`
12.0%, `sleep_duration` 11.0%, `sleep_quality` 8.5%, `calorie_expenditure` 7.7%. Missingness is mostly
uninformative **except `bmi`**: when `bmi` is missing, `unhealthy` share drops −5.4pp and `at-risk`/`fit`
rise (+3.0/+2.4pp). `heart_rate` and `diet_type` missingness show a smaller shift. → the **`missflag`
feature group is worth testing** (especially the bmi indicator), but expect a small effect.

## 5. Feature ↔ target signal
Mutual information ranks **`sleep_duration` (0.153)** and **`stress_level` (0.143)** far ahead, then
`physical_activity_level` (0.057), `bmi` (0.033), `step_count`/`exercise_duration` (~0.023). Weakest:
`gender` and `diet_type` (~0.0002, essentially noise). Single-feature balanced-accuracy lift agrees:
`stress_level` 0.754, `sleep_duration` 0.665, `physical_activity_level` 0.550, then a cluster of
mid-signal features; `gender`/`diet_type`/`water_intake`/`heart_rate` barely beat the floor.

## Modeling implications
1. **StratifiedKFold(5, seed 42)** — CV should track the LB (no drift). Confirmed default in `src/cv.py`.
2. **Keep NaNs**; GBDTs split on them natively. Add the **`missflag`** group as a cheap FE test (bmi
   indicator most promising). Interaction/ratio features (`ratio` group) are a secondary test.
3. **Decision tuning is essential** — raw argmax underperforms tuned balanced accuracy by ~3 points on
   the sample. Always report/compare the tuned OOF score.
4. `gender` and `diet_type` carry almost no signal; not worth engineering around.
